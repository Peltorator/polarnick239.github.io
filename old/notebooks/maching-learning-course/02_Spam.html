На эффективность дерева это не повлияло, и это логично, потому что дерево выбирает по какой границе отдельно взятого параметра сделать логическое разбиение, поэтому масштаб не важен - вместе с ним граница двигается без проблем.
В случае же KNN мы взяли евклидову метрику, а значит значения параметров учитываются одновременно, но тогда например в случае когда есть параметр с диапазоном от 10^6 до 10^7 и есть параметр от 0 до 1, то этот второй параметр не будет давать никакого вклада, он просто станет незаметен в евклидовой метрике за первым параметром. Если же диапазоны выровнять, то все параметры равноправны. Наверное чтобы быть устойчивыми к выбросам стоит выравнивать диапазоны не просто по min-max, а например по percentile 10%-90%.

Какой классификатор имеет AUC-ROC близкий к 0?
 - Когда TPR = 0, т.е. мало положительных примеров распозналось как положительные(неположительные примеры при этом можно распознавать положительно, т.к. это другая метрика - FPR).Например в точке с абсциссой FPR = 1.0 мы распознали всех, кто на самом деле быть распознан не должен был, а т.к. TPR при это = 0.0, то мы ни распознали ни одного из тех, кого стоило распознать. А значит это идеальный классификатор, т.к. его логику можно инвертировать (считать, что его распознавание=нераспознавание и наоборот), и получить идеальный классификатор.
Верно ли, что классификаторы из предыдущей части действуют путем сравнения некоторой оценки принадлежности положительному классу с порогом? Что это за оценки принадлежности? Какие пороги были выбраны?
 - KNN: да, верно, т.к. мы учитывали N соседей, то значение уверенности в том, что пример положительный имело значениемодно из {0/N, 1/N, ..., N/N}. Порогом положительного срабатывания выбрано 0.5 (включительно).
 - Деревья: скорее нет, т.к. каждый объект придет в конкретный лист, где ответ дается однозначно на основе какого-то из параметров. Но наверное можно сказать, что есть сравнение оценки принадлежности, считая что сравнение с порогом значения очередного параметра на развилке вносит вклад в вероятность положительной классификации.Можно смотреть на последнее разветвление, и уверенностью считать долю samples попавших в тот вариант, уверенность которого интересна.

def process(clf, X_data, name):
    global max_recall, max_recall_info
    results = clf.predict_proba(X_data)
    fpr, tpr, thresholds = roc_curve(Y[test], results[:, 1])
    auc_area = auc(fpr, tpr)
    ax1.plot(fpr, tpr, label=name+' (AUC-ROC={:.2f})'.format(auc_area))
    
    precisions = np.zeros_like(thresholds)
    recalls = np.zeros_like(thresholds)
    for i, threshold in enumerate(thresholds):
        y_pred = (results[:, 1] >= threshold)
        TP = ((Y[test] == 1) & (y_pred == 1)).sum()
        FP = ((Y[test] == 0) & (y_pred == 1)).sum()
        FN = ((Y[test] == 1) & (y_pred == 0)).sum()
        # Precision = TP / (TP + FP + 1) — точность, доля спама среди писем, классифицированных как спам
        # Recall = TP / (TP + FN + 1) — полнота, доля спама, которая была отфильтрована
        precisions[i] = TP / (TP + FP + 1)
        recalls[i] = TP / (TP + FN + 1)
    precisions[0] = 1
    recalls[0] = 0
    
    candidate = recalls[precisions >= min_precision].max()
    if candidate >= max_recall:
        max_recall = candidate
        threshold = thresholds[recalls[precisions >= min_precision].argmax()]
        max_recall_info = name + ' with threshold={:.2f}'.format(threshold)
    ax2.plot(recalls, precisions, label=name)

def calculate_auc_roc(clf, X, Y):
    results = clf.predict_proba(X)
    fpr, tpr, thresholds = roc_curve(Y, results[:, 1])
    auc_area = auc(fpr, tpr)
    return auc_area
